# Gemini Enterprise Document Parser

This application provides an automated end-to-end pipeline to ingest clinical or scientific documents into a Google Cloud Discovery Engine (Vertex AI Search) Data Store utilizing the advanced **LayoutParser** feature. It leverages **Gemini** to perform complex entity extraction (like understanding figures and tables) and augments the original unstructured documents with rich structural metadata.

## Workflow Pipeline

The ingestion process runs in multiple phases:

1. **Environment & Resource Setup:** 
   * Reads configurations from your `.env` file (shared with Sentinel or placed in `applications/sentinel/.env`).
   * Automatically provisions a new **Vertex AI Search Data Store** (`clinical-ds-...`) and an Enterprise **Search Engine** (`clinical-app-...`) if one isn't specified.
   * Auto-provisions Google Cloud Storage (GCS) buckets for source documents and staging if they don't exist.
2. **Phase 1: Datastore Ingestion**
   * Uploads raw documents (e.g., PDFs) from the `docs/` folder at the root of the repository to GCS.
   * Triggers an `ImportDocuments` job on the Data Store using the LayoutParser, which automatically chunks the data and recognizes structure (headings, tables, etc.).
3. **Phase 2: Complex Section Extraction with Gemini**
   * Retrieves the parsed chunks from the Datastore.
   * Scans for complex entities (e.g., tables or figures) and sends them to Gemini (e.g., `gemini-3.1-pro-preview`) for precise data extraction and summarization.
   * Appends these intelligent summaries back into the raw chunks.
4. **Phase 3: Final Update**
   * Uploads the finalized, enriched text blocks back to GCS and triggers a final `ImportDocuments` job to update the Datastore, providing a high-fidelity Search and QA experience.

## Quick Start

### Prerequisites

* Ensure your environment is configured via the root configuration script. The `.scripts/configure.sh` script automatically sets up the required `uv` virtual environment and installs all Python dependencies from the root `requirements.txt`.

```bash
# From the root of the repository
source .scripts/configure.sh
```

### Running the Ingestion

Place any unstructured documents (like PDFs) that you want to parse into the `docs/` folder located at the root of the repository.

Then, execute the main bash wrapper script:

```bash
.scripts/ingest_clinical_data.sh
```

This script orchestrates the pipeline, generates the resources, handles Python script execution, and updates your environment configurations with the created `DATA_STORE_ID`.

## Configuration Parameters

The pipeline uses the following environment variables (typically auto-generated by the bash script or manually configured in your `.env`):

* `PROJECT_ID`: Your Google Cloud Project ID.
* `DATA_STORE_ID`: The target Vertex AI Search Data Store ID (auto-created if left blank).
* `SOURCE_GCS_BUCKET`: The GCS bucket where raw `docs/` are uploaded (auto-created if left blank).
* `GEMINI_MODEL_NAME`: The Gemini model used for complex extractions (e.g., `gemini-3.1-pro-preview`).
* `REGION`: (Optional) The GCP region for your Vertex API requests, defaults to `us-central1`.
* `DISCOVERY_ENGINE_LOCATION`: Defaults to `us`.
